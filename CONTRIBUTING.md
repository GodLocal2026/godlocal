# Contributing to GodLocal

First off, thanks for taking the time to contribute! 🎉

The following is a set of guidelines for contributing to GodLocal.

## 🚀 Quick Start

1. Fork the repository
2. Clone your fork: `git clone https://github.com/YOUR_USERNAME/godlocal.git`
3. Create a branch: `git checkout -b feature/your-feature`
4. Make your changes
5. Commit: `git commit -m 'Add your feature'`
6. Push: `git push origin feature/your-feature`
7. Open a Pull Request

## 📝 Code Style

- **TypeScript** for all frontend/API code
- **Python** for backend services
- Use Prettier for formatting (auto-configured)
- Follow existing patterns in the codebase

## 🧪 Testing

```bash
# Run tests
npm test

# Run linting
npm run lint
```

## 🔀 Pull Request Process

1. Update the README.md if your change affects the documentation
2. Add tests for any new functionality
3. Ensure all tests pass
4. Use clear, descriptive commit messages
5. Reference any related issues in the PR description

## 💬 Communication

- **Issues** — Bug reports, feature requests
- **Discussions** — General questions, ideas
- **Pull Requests** — Code contributions

## 🎨 Types of Contributions

### 🐛 Bug Fixes
Found a bug? Open an issue first, then submit a PR with the fix.

### ✨ Features
Have an idea? Open an issue to discuss it before implementing.

### 📖 Documentation
Docs improvements are always welcome — no issue needed.

### 🌍 Translations
Help make GodLocal accessible to more people.

## 📋 Code of Conduct

Be respectful. Be constructive. We're all building something great together.
